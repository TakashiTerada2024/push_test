

# 開発サイクルについて

## 概要

アプリケーション開発のサイクルについて説明する

## 対象

Laravelアプリケーション開発担当者



# ワークフロー

git-flow。

ブランチ名は、Redmineチケット番号とする。

（ex: feature/1234 ,hotfix/9876）





## Commit前のチェックスクリプト

コミット前に以下のコマンドを実行し、エラーが発生した場合はコミット前にソースコードを修正する。

このコマンドが通らない状態のソースコードはコミットしない。

``` sh
composer qa-pre-commit
```

※psalmのinfoレベル警告は発生してもコミットして良い



## PR前のチェックスクリプト

プルリクエスト前に以下のコマンドを実行し、警告が発生した場合は修正等の対応を実施。

```sh
composer qa-all
```

- マージ担当者の承認を受けて、警告抑止のアノテーションを記述する
- 警告されている問題点を解消する

詳細については、[quality.md](./quality.md) 参照。
