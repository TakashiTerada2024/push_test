# GenerateSkipUrlControllerTest 仕様書

## 概要
GenerateSkipUrlControllerのユニットテスト仕様を定義する。

## テストケース

### 1. スキップURL生成の正常系
#### テストメソッド名
`test_スキップURLが正しく生成されること`

#### 前提条件
- 有効なリクエストデータが提供されている
- ApplicationSkipUrlServiceが正常に動作する

#### テストシナリオ
1. 必要なデータを含むリクエストを作成
2. コントローラーを実行
3. レスポンスを検証

#### 期待結果
- ステータスコード200が返却される
- レスポンスに以下の項目が含まれる：
  - success: true
  - message: 'スキップURLを生成しました'
  - data配列に以下の要素が含まれる：
    - url: 生成されたフルURL
    - ulid: 生成されたULID
    - apply_type_id: 申請種別ID
    - apply_type_name: 申請種別名
    - expired_at: フォーマットされた有効期限
    - text_to_copy: コピー用テキスト

### 2. サービス例外発生時のエラーハンドリング
#### テストメソッド名
`test_サービスで例外が発生した場合にエラーレスポンスが返却されること`

#### 前提条件
- サービスで例外が発生する状況が設定されている

#### テストシナリオ
1. サービスが例外をスローするようにモック設定
2. コントローラーを実行
3. エラーレスポンスを検証

#### 期待結果
- ステータスコード500が返却される
- エラーメッセージが含まれる
- success: falseが含まれる
